VM Network တွေကို **pfSense က NAT နဲ့ Host-Only ပေါင်းပြီး ချိတ်ဆက်ထားတာက အင်တာနက်ရဖို့ အကောင်းဆုံးနည်းလမ်းပါပဲ။**

pfSense ကို ဖွင့်ထားရင် သင့် Ubuntu Server နဲ့ Desktop နှစ်ခုလုံး အင်တာနက် ရပါလိမ့်မယ်။ ဒါပေမယ့် ဒီလိုအလုပ်လုပ်ဖို့အတွက် pfSense မှာ အောက်ပါအချက်တွေ မှန်ကန်စွာ ပြင်ဆင်ထားဖို့ လိုပါတယ်။

### ၁။ ဘာကြောင့် အလုပ်လုပ်တာလဲ (အခြေခံသဘောတရား)

သင့် ကွန်ရက်စနစ်က ပုံမှန် Home/Office Network နဲ့ အတူတူပါပဲ။

*   **pfSense** က Router/Gateway အဖြစ် အလုပ်လုပ်ပါတယ်။
*   **WAN (NAT)** ဆိုတာက သင့် Router ရဲ့ အင်တာနက်လိုင်းဝင်တဲ့ အပေါက် (ဥပမာ - Fiber ONT ကြိုးသွင်းတဲ့နေရာ) ဖြစ်ပါတယ် ။
*   **LAN (Host-Only)** ဆိုတာက သင့် Router ရဲ့ အတွင်းပိုင်း LAN Port တွေ (ဥပမာ - Switch ချိတ်ထားတဲ့နေရာ) ဖြစ်ပါတယ်။ ဒီမှာ သင့် Ubuntu Server နဲ့ Desktop တွေ ချိတ်ထားတာပါ။

ပုံမှန်အားဖြင့် pfSense ကို စအသုံးပြုတဲ့အခါ **Outbound NAT** က အလိုအလျောက် (Automatic Mode) ဖွင့်ပေးထားပါတယ် ။ ဆိုလိုတာက LAN (Host-Only) ကနေ WAN (NAT) ကိုဖြတ်ပြီး အင်တာနက်ကိုသွားမယ့် ကွန်ပျူတာတွေရဲ့ ကိုယ်ပိုင် IP လိပ်စာတွေကို pfSense ရဲ့ WAN IP လိပ်စာတစ်ခုတည်းနဲ့ အစားထိုးပြီး လမ်းကြောင်းပေးပို့ပေးတာပါ ။

ဒါကြောင့် သင့် Ubuntu တွေက အင်တာနက်ကိုသွားတဲ့အခါ "ကျုပ်တို့က 192.168.x.x က လာတာပါ" လို့ ပြောမသွားဘဲ pfSense ရဲ့ WAN IP နာမည်နဲ့ သွားလို့ အင်တာနက်က အသိအမှတ်ပြုပြီး ပြန်အကြောင်းကြားတာပါ။

### ၂။ အလုပ်မလုပ်ရင် ဘာတွေ စစ်ကြည့်ရမလဲ

တခါတလေ အလုပ်မလုပ်တာမျိုး ကြုံရင် အောက်ပါအချက်တွေကို စစ်ကြည့်ပါ။

#### A. pfSense ရဲ့ Outbound NAT ကို စစ်ဆေးခြင်း
pfSense မှာ Outbound NAT ကို မှားယွင်းပြီး ပိတ်ထားမိရင် (ဒါမှမဟုတ် Advanced Mode ပြောင်းထားမိရင်) အင်တာနက် အလုပ်မလုပ်တော့ပါဘူး ။
*   **သွားရမယ့်နေရာ** - pfSense Web GUI က `Firewall > NAT` ကိုသွားပါ။ `Outbound` Tab ကိုနှိပ်ပါ။
*   **စစ်ဆေးရန်** - **"Mode"** က **"Automatic Outbound NAT"** မှာ ရှိရပါမယ်။
*   **ပြင်ဆင်ရန်** - တခြား Mode ဖြစ်နေရင် Automatic ကိုရွေးပြီး Save လိုက်ပါ။

#### B. Firewall Rules (ခွင့်ပြုချက်စည်းမျဉ်းများ) ကို စစ်ဆေးခြင်း
Outbound NAT ရှိပေမယ့် LAN (Host-Only) ကနေ အင်တာနက်ကို ထွက်ခွင့်ပြုတဲ့ Firewall Rule မရှိရင်လည်း အလုပ်မလုပ်ပါဘူး ။
*   **သွားရမယ့်နေရာ** - `Firewall > Rules` ကိုသွားပါ။ **LAN** (သင်သတ်မှတ်ထားတဲ့ Host-Only Interface နာမည်) ကိုရွေးပါ။
*   **စစ်ဆေးရန်** - အောက်ပါအတိုင်း Rule တစ်ခု ရှိရပါမယ်။
    *   **Action**: `Pass` (ခွင့်ပြုမည်)
    *   **Protocol**: `Any` (သို့) `TCP/UDP`
    *   **Source**: `LAN subnet` (ဥပမာ - 192.168.1.0/24)
    *   **Destination**: `Any` (အင်တာနက်ပေါ်က ဘယ်နေရာမဆို)
*   **မရှိရင်** - Add လုပ်ပြီး အပေါ်ကလို သတ်မှတ်ပေးပါ။ များသောအားဖြင့် pfSense ကို စတင်တပ်ဆင်တဲ့အခါ ဒီ Rule ကို အလိုအလျောက် ဖန်တီးပေးပါတယ်။

#### C. Ubuntu ဘက်က စစ်ဆေးခြင်း (Gateway နဲ့ DNS)
Ubuntu တွေဘက်က ကိုယ့် Gateway (လမ်းကြောင်းပြဘို့) နဲ့ DNS (ဒိုမိန်းနာမည်ကို IP ပြန်ခေါ်ဖို့) ကို pfSense ဆီကို ညွှန်ပြထားရပါမယ်။
*   Ubuntu မှာ Network Settings ကိုဖွင့်ပြီး IPv4 Setting မှာ Method ကို **"Automatic (DHCP)"** ထားထားရင် pfSense က ဒါတွေကို အလိုအလျောက် ပေးပါလိမ့်မယ်။
*   ဒါမှမဟုတ် Manual ထားရင် Gateway နဲ့ DNS Server နေရာမှာ **pfSense ရဲ့ LAN IP** (ဥပမာ - 192.168.1.1) ကို ထည့်ထားရပါမယ်။

### ၃။ နောက်ထပ် သိထားသင့်တဲ့ အချက်လေး (NAT Reflection)

သင်က Ubuntu Server ပေါ်က Web Server (Port 80) ကို pfSense ရဲ့ WAN ကနေ လှမ်းကြည့်လို့ရအောင် Port Forwarding လုပ်ထားတယ် ဆိုပါစို့။
အဲဒီအခါ အင်တာနက်ကလူတွေ သင့် WAN IP ကိုသုံးပြီး ကြည့်လို့ရပေမယ့် သင့်အတွင်းပိုင်း ကွန်ရက် (ဥပမာ - Ubuntu Desktop) ကနေ WAN IP ကိုသုံးပြီး ကြည့်မယ်ဆိုရင် အလုပ်မလုပ်တာမျိုး ကြုံရတတ်ပါတယ် ။
ဒါကို ဖြေရှင်းဖို့ **NAT Reflection (သို့) Split DNS** ကို သုံးလေ့ရှိပါတယ်။
