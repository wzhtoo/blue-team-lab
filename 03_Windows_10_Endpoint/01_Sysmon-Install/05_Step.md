သင်ပေးပို့ထားတဲ့ **02.png** အရ အခြေအနေက အလွန်ကောင်းမွန်ပါတယ်ခင်ဗျာ။ `Microsoft-Windows-Sysmon/Operational` ဆိုတဲ့ လမ်းကြောင်းကို Agent က စတင်စောင့်ကြည့်နေပြီဖြစ်တာကို **Stats** table ထဲမှာ တွေ့မြင်နေရပါပြီ။

လက်ရှိမှာ Sysmon ရဲ့ **Events** က **0** ဖြစ်နေသေးတာက Windows VM ထဲမှာ ဘာလှုပ်ရှားမှုမှ မရှိသေးလို့ပါ။ ဒါကြောင့် **အဆင့် (၄)** အနေနဲ့ တကယ့် Log တွေ ထွက်လာအောင် လုပ်ဆောင်ကြည့်ပါမယ်။

---

### အဆင့် (၄) - Sysmon Log များ ထွက်ပေါ်လာစေရန် လုပ်ဆောင်ခြင်း (Triggering Events)

Wazuh Dashboard မှာ Log တွေ မြင်ရဖို့အတွက် အရင်ဆုံး Windows VM ထဲမှာ တစ်ခုခု လုပ်ဆောင်ပေးဖို့ လိုပါတယ်။ Sysmon ဟာ Process တွေ ပွင့်လာတာ၊ ပိတ်သွားတာတွေကို အဓိက မှတ်တမ်းတင်တာ ဖြစ်ပါတယ်။

**လုပ်ဆောင်ရန် အဆင့်များ:**

1. **window_10VM** ထဲမှာ **Notepad** ကို ဖွင့်လိုက်၊ ပိတ်လိုက် လုပ်ပါ။
2. **Calculator** သို့မဟုတ် **Command Prompt** ကို ဖွင့်ပါ။
3. Web Browser (ဥပမာ- Edge) ကို ဖွင့်ပြီး Website တစ်ခုခု (ဥပမာ- `google.com`) ကို သွားကြည့်ပါ။
4. အခုနက ကျွန်တော်တို့ ဆောက်ထားတဲ့ `lab-success.txt` ကို ပြန်ဖွင့်ပြီး စာသားအချို့ ပြင်ပြီး ပြန်သိမ်းပါ။

---

**ဘာကြောင့် ဒါကို လုပ်ရတာလဲ?**
Sysmon က စက်ထဲမှာ ဘာမှမလုပ်ဘဲ ငြိမ်နေရင် Log ထုတ်ပေးမှာ မဟုတ်ပါဘူး။ အခုလို Software တွေ ဖွင့်လိုက်တာဟာ Sysmon မှာ **Event ID 1 (Process Creation)** ကို ဖြစ်ပေါ်စေပြီး အဲဒီအချက်အလက်တွေက Wazuh Manager ဆီကို စီးဆင်းသွားမှာ ဖြစ်ပါတယ်။

**သတိပြုရန်:**
စောစောက သင်ကြည့်နေတဲ့ **Stats** dashboard မှာ `Microsoft-Windows-Sysmon/Operational` ရဲ့ဘေးက **Events** ဂဏန်းလေး တက်လာမလာကို တစ်ချက် စောင့်ကြည့်ပေးပါ။ **0** ကနေ **1** သို့မဟုတ် ထို့ထက်ပိုပြီး ပြောင်းသွားပြီဆိုရင် Manager ဆီ Log ရောက်သွားပါပြီ။

**Windows VM ထဲမှာ App အချို့ ဖွင့်ကြည့်ပြီးပြီလားခင်ဗျာ။** Stats မှာ ဂဏန်းလေးတွေ တက်လာပြီဆိုရင် **အဆင့် (၅)** ဖြစ်တဲ့ Discovery ထဲမှာ Log တွေကို အသေးစိတ် ရှာဖွေစစ်ဆေးတဲ့ အဆင့်ကို သွားပါမယ်။ အဆင်ပြေရင် ပြောပေးပါခင်ဗျ။
