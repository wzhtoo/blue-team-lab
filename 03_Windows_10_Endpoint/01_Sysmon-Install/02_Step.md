ဟုတ်ကဲ့ခင်ဗျာ၊ အဆင့် (၁) ဖြစ်တဲ့ Config ထဲမှာ Sysmon လမ်းကြောင်း ထည့်တာ ပြီးသွားပြီဆိုရင် **အဆင့် (၂)** ကတော့ အဲဒီပြောင်းလဲမှုတွေကို Agent က သိရှိသွားအောင် လုပ်ဆောင်ရမယ့် အဆင့်ဖြစ်ပါတယ်။

---

### အဆင့် (၂) - Wazuh Agent Service ကို Restart ချခြင်း

`ossec.conf` ဖိုင်ထဲမှာ အချက်အလက်သစ်တွေ ထည့်ပြီးရုံနဲ့ Agent က ချက်ချင်း အလုပ်မလုပ်သေးပါဘူး။ Agent ကို ပိတ်ပြီး ပြန်ဖွင့်မှသာ (Restart ချမှသာ) Sysmon Log တွေကို စတင်ဖတ်ယူမှာ ဖြစ်ပါတယ်။

**လုပ်ဆောင်ရန် အဆင့်များ:**

1. **window_10VM** ထဲက **PowerShell** ကို **Administrator** အဖြစ် ဖွင့်ထားဆဲ ဖြစ်ရပါမယ်။
2. အောက်ပါ Command ကို ရိုက်ထည့်ပြီး Enter ခေါက်ပါ-
```powershell
Restart-Service -Name "WazuhSvc"

```


3. Command ရိုက်ပြီးလို့ Error ဘာမှမတက်ဘဲ နောက်စာကြောင်းအသစ် ဆင်းသွားရင် အောင်မြင်ပါတယ်။

---

**ဘာကြောင့် ဒါကို လုပ်ရတာလဲ?**
Wazuh Agent (WazuhSvc) ဟာ စတင်ပွင့်လာတဲ့ အချိန်မှာပဲ `ossec.conf` ထဲက သတ်မှတ်ချက်တွေကို ဖတ်တာဖြစ်လို့၊ ကျွန်တော်တို့ အသစ်ထည့်လိုက်တဲ့ Sysmon လမ်းကြောင်းကို သိရှိသွားအောင် Service ကို ပြန်နိုးပေးရတာ ဖြစ်ပါတယ်။

**သတိပြုရန်:**
အကယ်၍ စောစောကလို `Cannot stop WazuhSvc` ဆိုတဲ့ Error တက်ခဲ့ရင် စိတ်မပူပါနဲ့၊ နောက်တစ်ကြိမ် ထပ်ရိုက်ပေးပါ။ နောက်ဆုံးမှာ `Get-Service WazuhSvc` လို့ ရိုက်ကြည့်တဲ့အခါ Status မှာ **Running** လို့ ပြနေရင် အဆင်ပြေပါပြီ။

**ဒီအဆင့် (၂) ပြီးသွားပြီဆိုရင် အဆင့် (၃) ဖြစ်တဲ့ Dashboard မှာ Agent Status ပြန်စစ်ဆေးတဲ့အဆင့်ကို ဆက်သွားပါမယ်။ အဆင်ပြေလားခင်ဗျာ။**
