ဟုတ်ကဲ့ခင်ဗျာ၊ အဆင့် (၂) မှာ Service ကို အောင်မြင်စွာ Restart ချပြီးပြီဆိုရင် **အဆင့် (၃)** ကတော့ Manager နဲ့ Agent ကြား အဆက်အသွယ် ပြန်ရမရ အတည်ပြုတဲ့ အဆင့်ဖြစ်ပါတယ်။

---

### အဆင့် (၃) - Wazuh Dashboard တွင် Agent Status ကို ပြန်လည်စစ်ဆေးခြင်း

ကျွန်တော်တို့ Config တွေ ပြင်ဆင်ပြီး Restart ချလိုက်တဲ့အတွက် Agent ဟာ ခဏတာ ပျောက်သွားနိုင်ပါတယ်။ အဆင့် (၄) ဖြစ်တဲ့ Log တွေ စောင့်ကြည့်တဲ့ အဆင့်ကို မသွားခင် Agent က **Active** ဖြစ်နေဖို့ လိုအပ်ပါတယ်။

**လုပ်ဆောင်ရန် အဆင့်များ:**

1. Ubuntu Machine ထဲက Browser မှာ **Wazuh Dashboard** ကို ဖွင့်ပါ။
2. ဘယ်ဘက်အပေါ်ဒေါင့်က Menu (စင်းချောင်း ၃ ချောင်း) ကို နှိပ်ပြီး **Home** သို့မဟုတ် **Endpoints > Agents** ကို သွားပါ။
3. သင်၏ **window_10VM (ID: 003)** ၏ Status ကို ကြည့်ပါ။
4. Status နေရာမှာ **Active** (အစိမ်းရောင်စက်လေး) ပြနေပြီဆိုရင် ချိတ်ဆက်မှု အောင်မြင်ပါပြီ။

---

**ဘာကြောင့် ဒါကို လုပ်ရတာလဲ?**
အကယ်၍ `ossec.conf` ထဲမှာ စာလုံးပေါင်းမှားတာ (Syntax error) ရှိနေရင် Agent က `Active` ပြန်မဖြစ်လာတော့ပါဘူး။ ဒါကြောင့် Status ကို အရင်စစ်တာဟာ နောက်ပိုင်း အဆင့်တွေမှာ Log မတက်တဲ့ပြဿနာ မဖြစ်အောင် ကြိုတင်ကာကွယ်တာ ဖြစ်ပါတယ်။

**သတိပြုရန်:**
စက် Status က `Never connected` သို့မဟုတ် `Disconnected` ဖြစ်နေရင်တော့ အဆင့် (၁) က Config စာသားလေးတွေကို ပြန်စစ်ဖို့ လိုအပ်ပါလိမ့်မယ်။

**Dashboard မှာ Agent လေး Active ပြန်ဖြစ်နေပြီလားခင်ဗျာ။** အဆင်ပြေရင် **အဆင့် (၄)** ဖြစ်တဲ့ Sysmon Log တွေကို ရှာဖွေကြည့်ရှုတဲ့ Discovery အဆင့်ကို ဆက်သွားပါမယ်။
