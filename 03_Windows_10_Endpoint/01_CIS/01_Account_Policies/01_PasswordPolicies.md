# CIS Password Policy သတ်မှတ်နည်း အဆင့်ဆင့်

ပထမဦးဆုံး `Win + R` ကိုနှိပ်ပြီး `gpedit.msc` လို့ ရိုက်ထည့်ကာ Group Policy Editor ကို ဖွင့်ပါ။ ပြီးနောက် အောက်ပါလမ်းကြောင်းအတိုင်း သွားပါ-

> **Computer Configuration** > **Windows Settings** > **Security Settings** > **Account Policies** > **Password Policy**

အဲဒီထဲမှာ CIS က အဓိက အကြံပြုထားတဲ့ အချက် ၆ ချက်ရှိပါတယ်-

---

### ၁။ Enforce password history

* **CIS အကြံပြုချက်:** `24` သို့မဟုတ် ထိုထက်ပိုသော password အဟောင်းများကို မှတ်ထားရန်။
* **ရည်ရွယ်ချက်:** အသုံးပြုသူက password အသစ်လဲတဲ့အခါ အရင်သုံးခဲ့ဖူးတဲ့ password ဟောင်းတွေကိုပဲ ထပ်ခါထပ်ခါ ပြန်မသုံးနိုင်အောင် တားဆီးဖို့ ဖြစ်ပါတယ်။

### ၂။ Maximum password age

* **CIS အကြံပြုချက်:** `365` ရက် သို့မဟုတ် ထိုထက်နည်းသော ရက်ပေါင်း (ပုံမှန်အားဖြင့် ရက် ၉၀ သို့မဟုတ် တစ်နှစ်တစ်ကြိမ်)။
* **ရည်ရွယ်ချက်:** Password တစ်ခုကို အကြာကြီး သုံးမနေစေဘဲ ပုံမှန် အသစ်လဲလှယ်ခိုင်းခြင်းဖြင့် ခိုးယူခံရနိုင်ခြေကို လျှော့ချဖို့ ဖြစ်ပါတယ်။

### ③ Minimum password age

* **CIS အကြံပြုချက်:** `1` ရက်။
* **ရည်ရွယ်ချက်:** Password ကို အခုချက်ချင်းလဲပြီး ချက်ချင်း password ဟောင်းဆီ ပြန်မပြောင်းနိုင်အောင် (Password history ကို ကျော်မတက်နိုင်အောင်) အနည်းဆုံး တစ်ရက်ကြာမှ ထပ်လဲခွင့်ပေးတာ ဖြစ်ပါတယ်။

### ၄။ Minimum password length

* **CIS အကြံပြုချက်:** အနည်းဆုံး `14` character သို့မဟုတ် ထိုထက်ပို၍ ထားရန်။
* **ရည်ရွယ်ချက်:** Password ပိုရှည်လေလေ Brute-force attack နဲ့ ခန့်မှန်းဖို့ ပိုခက်လေလေ ဖြစ်တာကြောင့် အရှည်ကို အလေးထားတာ ဖြစ်ပါတယ်။

### ၅။ Password must meet complexity requirements

* **CIS အကြံပြုချက်:** `Enabled` (ဖွင့်ထားရန်)။
* **ရည်ရွယ်ချက်:** Password ထဲမှာ စာလုံးကြီး၊ စာလုံးသေး၊ ဂဏန်းနဲ့ သင်္ကေတ (Symbols) တွေ ပါဝင်အောင် အတင်းအကျပ် သတ်မှတ်တာ ဖြစ်ပါတယ်။

### ၆။ Store passwords using reversible encryption

* **CIS အကြံပြုချက်:** `Disabled` (ပိတ်ထားရန်)။
* **ရည်ရွယ်ချက်:** Password တွေကို မူလအတိုင်း ပြန်ဖတ်လို့ရမယ့် ပုံစံနဲ့ သိမ်းဆည်းတာကို ပိတ်ထားဖို့ ဖြစ်ပါတယ်။ ဒါကို ဖွင့်ထားရင် ဟက်ကာတွေက password ကို အလွယ်တကူ ပြန်ဖတ်လို့ ရသွားနိုင်ပါတယ်။

---

### အနှစ်ချုပ် CIS Policy ဇယား

| Policy အမည် | CIS အကြံပြုချက် (Standard) |
| --- | --- |
| Enforce password history | 24 passwords remembered |
| Maximum password age | 365 days or less |
| Minimum password age | 1 day |
| Minimum password length | 14 or more characters |
| Password complexity | Enabled |
| Reversible encryption | Disabled |

**မှတ်ချက်:** CIS Benchmark တွေက အမြဲ Update ဖြစ်နေတတ်သလို လုပ်ငန်းခွင် အခြေအနေပေါ်မူတည်ပြီး အပြောင်းအလဲ ရှိနိုင်ပါတယ်။ ဒါပေမဲ့ အထက်ပါ အချက်တွေကတော့ အခြေခံအကျဆုံးနဲ့ အရေးကြီးဆုံး လုံခြုံရေး စံနှုန်းတွေ ဖြစ်ပါတယ်ခင်ဗျာ။

ဟုတ်ကဲ့ပါ၊ အခုနက Password Policy အပြီးမှာ လုံခြုံရေးအတွက် နောက်ထပ် အရေးကြီးတဲ့ **Account Lockout Policy** အကြောင်းကို CIS (Center for Internet Security) စံနှုန်းတွေနဲ့ ဆက်ပြီး ရှင်းပြပေးပါ့မယ်။

Account Lockout ဆိုတာကတော့ Brute-force (ခန့်မှန်း password များစွာကို တရစပ်ရိုက်ခြင်း) တိုက်ခိုက်မှုတွေကို တားဆီးဖို့အတွက် Password အကြိမ်ကြိမ် မှားရိုက်ရင် အဲဒီအကောင့်ကို ခေတ္တပိတ်ပစ်တဲ့ စနစ်ဖြစ်ပါတယ်။

---

# CIS Account Lockout Policy သတ်မှတ်ပုံ

`gpedit.msc` ထဲက လမ်းကြောင်းကတော့ အရင်အတိုင်းပါပဲ-

> **Computer Configuration** > **Windows Settings** > **Security Settings** > **Account Policies** > **Account Lockout Policy**

အဲဒီအောက်မှာ အဓိက သတ်မှတ်ရမယ့် အချက် ၃ ချက် ရှိပါတယ်-

#### ၁။ Account lockout duration (အကောင့် ပိတ်ထားမည့် ကြာချိန်)

* **CIS အကြံပြုချက်:** `15` မိနစ် သို့မဟုတ် ထိုထက်ပို၍ ထားရန်။
* **ရည်ရွယ်ချက်:** Password ခန့်မှန်းသူ တစ်ယောက်က အကောင့်ကို ပိတ်မိသွားအောင် လုပ်လိုက်ရင် အဲဒီလူ ထပ်ပြီး စမ်းသပ်လို့မရအောင် အချိန်တစ်ခုအထိ စောင့်ခိုင်းတာ ဖြစ်ပါတယ်။ `0` လို့ ထားရင်တော့ Administrator က လာဖွင့်ပေးမှသာ ပြန်ပွင့်မှာ ဖြစ်ပါတယ်။

#### ၂။ Account lockout threshold (Password အမှား ခံနိုင်ရည် အကြိမ်ရေ)

* **CIS အကြံပြုချက်:** `5` သို့မဟုတ် ထိုထက်နည်းသော အကြိမ်ရေ (ဥပမာ- ၅ ကြိမ်မှားရင် ပိတ်မယ်)။
* **ရည်ရွယ်ချက်:** လူတစ်ယောက်က သူ့ password သူ မေ့နေလို့ မှားရိုက်မိတာမျိုးကို ခွင့်ပြုပေးနိုင်သလို၊ ဟက်ကာက အကြိမ်ပေါင်းများစွာ တရစပ် စမ်းနေတာကိုလည်း တားဆီးနိုင်ဖို့အတွက် ၅ ကြိမ်ဆိုတာ သင့်တော်တဲ့ စံနှုန်းဖြစ်ပါတယ်။

#### ၃။ Reset account lockout counter after (မှားယွင်းမှု မှတ်တမ်းကို ပြန်ဖျက်မည့် ကြာချိန်)

* **CIS အကြံပြုချက်:** `15` မိနစ် သို့မဟုတ် ထိုထက်ပို၍ ထားရန်။
* **ရည်ရွယ်ချက်:** ဒါကတော့ ပထမအကြိမ် မှားရိုက်မိတာနဲ့ ဒုတိယအကြိမ် မှားရိုက်မိတာကြားက အချိန်ကို မှတ်ထားတာပါ။ ဥပမာ- ၁၅ မိနစ်အတွင်း ၅ ကြိမ်ပြည့်အောင် မှားမှသာ အကောင့်ကို ပိတ်မှာ ဖြစ်ပါတယ်။ ၁၅ မိနစ်ကျော်သွားရင်တော့ အမှားရေတွက်မှုကို သုညကနေ ပြန်စမှာ ဖြစ်ပါတယ်။

---

### CIS Standard Summary Table

| Policy အမည် | CIS အကြံပြုချက် (Standard) |
| --- | --- |
| Account lockout duration | 15 or more minutes |
| Account lockout threshold | 5 or fewer invalid logon attempts |
| Reset lockout counter after | 15 or more minutes |

---

### ဒါတွေကို ဘာလို့ သတ်မှတ်ရတာလဲ?

လုပ်ငန်းခွင်တွေမှာ CIS Benchmark တွေကို သုံးရခြင်းက အကောင့်တွေကို Brute-force attack တွေကနေ ကာကွယ်ရုံတင်မကဘဲ၊ လုံခြုံရေးပိုင်းဆိုင်ရာ အသိအမှတ်ပြုလက်မှတ် (Compliance) တွေရဖို့အတွက်လည်း အထောက်အကူပြုလို့ ဖြစ်ပါတယ်။

# UAC (User Account Control)

Windows ရဲ့ လုံခြုံရေး အလွှာတစ်ခုဖြစ်ပြီး Malware တွေ ဒါမှမဟုတ် ခွင့်ပြုချက်မရှိတဲ့ Software တွေက OS ရဲ့ အရေးကြီးတဲ့နေရာတွေကို ပြောင်းလဲပစ်တာမျိုးမလုပ်နိုင်အောင် တားဆီးပေးတဲ့ Firewall အသေးစားလေးတစ်ခုလို့ တင်စားနိုင်ပါတယ်။

UAC ကို ပိတ်သင့်၊ မပိတ်သင့် ငြင်းခုန်နေကြတဲ့အချက်တွေကို CIS (Center for Internet Security) စံနှုန်းတွေနဲ့ ချိန်ထိုးပြီး ရှင်းပြပေးပါ့မယ်။

---

### UAC ဆိုတာ ဘာလဲ?

ပုံမှန်အားဖြင့် Windows မှာ Admin အကောင့်နဲ့ သုံးနေရင်တောင် Browser ဖွင့်တာ၊ သီချင်းနားထောင်တာမျိုးမှာ UAC က အနောက်မှာ ငြိမ်နေပါလိမ့်မယ်။ ဒါပေမဲ့ System Setting ပြောင်းတာ ဒါမှမဟုတ် Software တစ်ခု သွင်းတာမျိုး (Admin အခွင့်အရေး လိုအပ်လာတဲ့အခါ) မှာ **"Do you want to allow this app to make changes to your device?"** ဆိုပြီး Screen က မှောင်သွားပြီး မေးခွန်းတက်လာပါလိမ့်မယ်။ အဲဒါ UAC ပါပဲ။
---

### CIS Policy အရ UAC သတ်မှတ်ချက်များ

CIS Benchmark တွေမှာ UAC ကို ပိတ်ထားဖို့ ဘယ်တော့မှ အကြံမပြုပါဘူး။ အဓိက သတ်မှတ်ရမယ့် Policy တွေကတော့-

* **User Account Control: Admin Approval Mode for the Built-in Administrator account:**
* **CIS အကြံပြုချက်:** `Enabled`
* **ရည်ရွယ်ချက်:** Windows ရဲ့ ပင်ကိုယ် Admin (Built-in Administrator) ကို သုံးရင်တောင် UAC က မေးခွန်းမေးဖို့ ဖြစ်ပါတယ်။


* **User Account Control: Behavior of the elevation prompt for administrators:**
* **CIS အကြံပြုချက်:** `Prompt for consent on the secure desktop`
* **ရည်ရွယ်ချက်:** Screen ကို မှောင်သွားစေပြီး (Secure Desktop) ခွင့်ပြုချက်တောင်းဖို့ ဖြစ်ပါတယ်။ ဒါမှ Malware တွေက Background ကနေ 'Yes' ကို အလိုအလျောက် နှိပ်သွားတာမျိုး မလုပ်နိုင်မှာပါ။


* **User Account Control: Run all administrators in Admin Approval Mode:**
* **CIS အကြံပြုချက်:** `Enabled`
* **ရည်ရွယ်ချက်:** ဒါဟာ UAC ရဲ့ အသက်ပါပဲ။ ဒါကို ပိတ်လိုက်ရင် UAC က လုံးဝ အလုပ်မလုပ်တော့ပါဘူး။
---

### UAC ပိတ်ခြင်း vs ဖွင့်ခြင်း (ငြင်းခုန်မှုများ)

#### ၁။ ပိတ်ထားချင်သူများ၏ အမြင် (The Convenience Argument)

* **အနှောင့်အယှက်ဖြစ်ခြင်း:** Software တစ်ခုခု ဖွင့်တိုင်း၊ Setting ပြောင်းတိုင်း ခဏခဏ မေးနေတာ စိတ်ရှုပ်ဖို့ကောင်းတယ်လို့ ယူဆကြပါတယ်။
* **Legacy Software:** တစ်ချို့သော ဆော့ဝဲလ်အဟောင်းတွေက UAC ဖွင့်ထားရင် အလုပ်ကောင်းကောင်းမလုပ်တာမျိုး ရှိတတ်ပါတယ်။
* **Advanced User:** "ငါက ဘာနှိပ်ရမလဲ သိတယ်၊ Malware မဝင်အောင် ဆင်ခြင်နိုင်တယ်" ဆိုတဲ့ ယုံကြည်မှုကြောင့် ပိတ်တတ်ကြပါတယ်။

#### ၂။ ဖွင့်ထားသင့်သူများ၏ အမြင် (The Security Argument - CIS Standard)

* **Silent Malware Prevention:** Malware တော်တော်များများဟာ User မသိအောင် Background ကနေ System ဖိုင်တွေကို ပြောင်းလဲပစ်ဖို့ ကြိုးစားကြပါတယ်။ UAC ဖွင့်ထားရင် အဲဒီအချိန်မှာ Prompt တက်လာမှာဖြစ်လို့ User က ရိပ်မိသွားပါလိမ့်မယ်။
* **Ransomware ကာကွယ်ခြင်း:** ဖိုင်တွေကို Lock ချမယ့် Ransomware တွေဟာ Admin အခွင့်အရေးကို လိုချင်ကြပါတယ်။ UAC က အဲဒါကို နောက်ဆုံးအဆင့်ကနေ တားပေးတာပါ။
* **Accidental Mistakes:** User ကိုယ်တိုင် မှားယွင်းပြီး ဖျက်မိနိုင်တဲ့ System settings တွေကို တစ်ခါပြန်စဉ်းစားဖို့ အချိန်ရစေပါတယ်။
---

### ကျနော့်ရဲ့ အကြံပြုချက် (Peers' Advice)

လုံခြုံရေးကို ဦးစားပေးတဲ့ **Cyber Defense** ရှုထောင့်ကကြည့်ရင် UAC ကို လုံးဝ **မပိတ်သင့်ပါဘူး**။ အကယ်၍ ခဏခဏ မေးတာ စိတ်ညစ်ဖို့ကောင်းတယ်ဆိုရင်တောင် "Never Notify" အဆင့်ထိ မချဘဲ အလယ်အလတ်အဆင့်မှာ ထားသင့်ပါတယ်။

ဒါပေမဲ့ CIS စံနှုန်းအတိုင်း အမြင့်ဆုံးလုံခြုံရေးလိုချင်ရင်တော့ **"Always Notify"** မှာ ထားတာက အကောင်းဆုံးဖြစ်ပါတယ်ခင်ဗျာ။
